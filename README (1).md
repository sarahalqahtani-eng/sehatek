# 🏥 صحتك — Sehatak
### تطبيق مراقبة صحية ذكي | Smart Health Monitoring App

---

## 📋 نظرة عامة

**صحتك** هو تطبيق ويب عربي لمراقبة الصحة الشخصية، يجمع بين بيانات السوارة الذكية والتحاليل الطبية والذكاء الاصطناعي لتقديم تجربة صحية متكاملة.

**التقنيات المستخدمة:** HTML5 · CSS3 · JavaScript · Supabase (Auth + Database + Storage + Edge Functions) · OpenAI API

## ✨ الميزات الرئيسية

### 1. 🔐 نظام المصادقة (Authentication)
- تسجيل حساب جديد مع بيانات صحية (طول، وزن، فصيلة دم، هدف صحي)
- تسجيل دخول بالإيميل والباسورد عبر Supabase Auth
- حساب تلقائي لـ BMI و BMR والسعرات اليومية عند التسجيل
- حفظ بيانات الجلسة للتنقل بين الصفحات

### 2. ⌚ السوارة الذكية (Wearable Simulator)
محاكاة كاملة لسوارة ذكية تظهر بجانب الداشبورد:
- **LED Indicator:** أزرق = غير متصل، أخضر متوهج = متصل
- **شاشة حية** تعرض: نبض القلب، السكر، الضغط، الحرارة، الخطوات
- **تحديث كل 4 ثواني** بقراءات تعتمد على فورمولات طبية حقيقية
- **حفظ تلقائي** بقاعدة البيانات كل 30 ثانية
- **كشف الأوضاع:** نوم 😴 / نشاط 🏃 / توتر 😰
- **تنبيهات صحية:** تسارع القلب، ارتفاع الضغط، خطورة السكر

### 3. 📐 الفورمولات الطبية (Mathematical Formulation)

| المعادلة | الوصف |
|----------|-------|
| `BMI = W / H²` | مؤشر كتلة الجسم |
| `α = 1 + 0.02 × (BMI − 22)` | معامل التكيّف الفسيولوجي |
| `HR_base = 75 × α` | نبض القلب الأساسي |
| `BP_base = 110 × α` | ضغط الدم الأساسي |
| `S_sleep = 1 if HR ≤ HR_base-10 AND ST ≤ 2` | كشف وضع النوم |
| `S_active = 1 if ST ≥ 15` | كشف النشاط البدني |
| `S_stress = 1 if HR ≥ HR_base+25 AND !active` | كشف التوتر |
| `A_HR = 1 if HR ≥ 120×α AND !active` | تنبيه تسارع القلب |
| `A_BP = 1 if BP ≥ 130×α` | تنبيه ارتفاع الضغط |
| `A_GL = 1 if GL ≥ 140 OR GL ≤ 70` | تنبيه السكر |

### 4. 🧪 نظام التحاليل (Lab Reports)
- رفع ملفات CSV تحتوي نتائج التحاليل
- تصنيف تلقائي: طبيعي ✅ / مرتفع 🔴 / منخفض 🟡
- نصائح طبية مخصصة حسب كل تحليل
- حذف التحاليل مع تأكيد داخل التطبيق

**صيغة CSV المدعومة:**
```csv
test_name,value,unit,ref_low,ref_high
HbA1c,5.7,%,4.0,5.6
Cholesterol,185,mg/dL,0,200
Vitamin D,28,ng/mL,30,100
Hemoglobin,12,g/dL,13.5,17.5
```

**النصائح الطبية المبرمجة:**

| التحليل | الحالة | النصائح |
|---------|--------|---------|
| Vitamin D | منخفض | شمس 15-20 دقيقة، أطعمة غنية بفيتامين D، مكملات |
| HbA1c | مرتفع | تقليل السكر، نشاط يومي، متابعة طبية |
| Cholesterol | مرتفع | تقليل دهون مشبعة، ألياف، مشي يومي |
| Hemoglobin | منخفض | أطعمة غنية بالحديد، فيتامين C، فحص Ferritin |

### 5. 📊 التقارير الصحية (AI Reports)
- تقارير مولّدة بالذكاء الاصطناعي عبر Edge Function + OpenAI
- تعرض: ملخص الحالة + مستوى الخطورة (منخفض/متوسط/مرتفع) + نصائح
- تنحفظ مع إمكانية العرض والحذف
- صفحة عرض مخصصة بتصميم احترافي

### 6. 📅 التقويم والمواعيد (Calendar)
- تقويم شهري مع نقاط خضراء على الأيام اللي فيها مواعيد
- إضافة مواعيد طبية وأدوية يدوياً
- **اختيار الوقت بسكرولنق** (ساعة + دقيقة + ص/م)
- حذف المواعيد بزر السلة
- **مساعد ذكي يفهم النص العربي:**
  - `"موعد طبيب يوم 6/2 الساعة 5 مساء"` ✅
  - `"دواء الضغط يوم ١٥ مارس الساعه ٨ صباح"` ✅
  - `"حط لي تحليل دم 20/3 الساعة 10"` ✅

### 7. 👤 البروفايل الشخصي
- عرض: اسم، عمر، طول، وزن، BMI، سعرات، فصيلة دم، جنس، هدف، نشاط
- تعديل كل البيانات مع إعادة حساب BMI والسعرات تلقائياً
- الوصول من الداشبورد (ضغط الصورة/الاسم) أو الإعدادات

## 🗄️ قاعدة البيانات

| الجدول | الوصف | الأعمدة الرئيسية |
|--------|-------|------------------|
| `patient_profile` | بيانات المريض | user_id, first_name, weight_kg, height_cm, bmi, bmr |
| `wearable_data` | قراءات السوارة | heart_rate, blood_sugar, bp_systolic, stress_level |
| `lab_reports` | ملفات التحاليل | file_name, file_url, file_type |
| `lab_results` | نتائج التحاليل | test_name, value, unit, status, reference_min/max |
| `ai_daily_reports` | تقارير AI | summary, risk_level, report_type |

## ⚙️ التشغيل

### المتطلبات
- حساب [Supabase](https://supabase.com) (مجاني)
- متصفح حديث (Chrome/Edge/Firefox)
- VS Code + Live Server (اختياري)

### خطوات الإعداد

**1.** أنشئ مشروع Supabase وانسخ الـ URL و anon key

**2.** حدّث `app.js`:
```javascript
const SUPABASE_URL  = 'https://YOUR_PROJECT.supabase.co';
const SUPABASE_ANON = 'YOUR_ANON_KEY';
```

**3.** شغّل `supabase_schema.sql` بالـ SQL Editor

**4.** عطّل تأكيد الإيميل: Authentication → Sign In / Providers → Email → Confirm email = OFF

**5.** افتح `onboarding.html` وابدأ!
